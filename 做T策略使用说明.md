# 📈 专注做T策略使用说明

## 🎯 策略简介

**专注做T策略**是一个专门针对日内做T（日内交易）的量化交易策略，主要特点：

- **日内交易**：专注于当日买入、当日卖出的日内交易
- **自动执行**：根据预设条件自动触发买入和卖出
- **风险控制**：包含资金管理和订单处理机制
- **详细日志**：完整的交易日志记录

## 📋 策略功能

### 核心功能
1. **自动买入**：当价格低于昨日收盘价时自动买入
2. **智能做T**：满足条件时自动卖出当日买入的股票
3. **资金管理**：控制总持仓成本不超过上限
4. **订单处理**：处理部分成交和未完成订单
5. **实时监控**：监控当日最低价和最高价

### 做T触发条件

#### 条件1：日内涨幅大于5%并回落1%
- **触发时机**：盘中任何时间
- **触发条件**：
  - 涨幅 > 5%
  - 从当日最高价回落 > 1%

#### 条件2：14:55涨幅大于2%
- **触发时机**：每日14:55
- **触发条件**：涨幅 > 2%

## 🛠️ 策略参数

### 可配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `g.security` | "159206.SZ" | 交易标的（卫星ETF） |
| `g.buy_amount` | 10000 | 每次买入目标金额（元） |
| `g.max_cost` | 200000 | 单只股票持仓成本上限（元） |
| `g.t_gain_threshold` | 0.05 | 做T涨幅阈值：5% |
| `g.t_pullback_threshold` | 0.01 | 做T回落阈值：1% |
| `g.t_1455_gain_threshold` | 0.02 | 14:55做T涨幅阈值：2% |

### 运行时变量

| 变量 | 说明 |
|------|------|
| `g.realized_pnl` | 累计落袋盈亏 |
| `g.pending_orders` | 待处理的订单 |
| `stock_info['total_cost']` | 累计买入成本 |
| `stock_info['avg_cost']` | 平均成本 |
| `stock_info['today_buy_amount']` | 当日买入量 |
| `stock_info['t_done_today']` | 当日是否已执行做T |

## 📊 策略逻辑流程

### 1. 初始化阶段
- 设置策略参数
- 初始化股票相关变量
- 打印初始化信息

### 2. 每日开始前
- 检查并处理昨日未完成的订单

### 3. 盘中交易
- **买入逻辑**：价格低于昨日收盘价且当日未买入过
- **做T逻辑**：
  - 条件1：涨幅>5%且回落>1%
  - 条件2：14:55涨幅>2%
- **订单处理**：处理部分成交情况

### 4. 收盘统计
- 计算总市值、盈亏、盈亏比例
- 显示累计落袋盈亏

## ⚠️ 注意事项

1. **市场风险**：股票价格波动可能导致亏损
2. **流动性风险**：部分股票可能存在流动性不足
3. **交易成本**：频繁交易会产生较多交易费用
4. **滑点风险**：实际成交价格可能与预期不同
5. **系统风险**：网络或系统故障可能影响交易
