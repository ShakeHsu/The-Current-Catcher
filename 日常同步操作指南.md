# GitHub日常同步操作指南

## 完整工作流程

### 场景：在新电脑上开始工作

#### 步骤1：从GitHub获取代码（首次）

```powershell
# 1. 打开PowerShell
# 2. 进入项目目录
cd "C:\Users\XU\Documents\trae_projects\The Current Catcher"

# 3. 获取最新代码
git pull origin main

# 4. 查看状态
git status
```

**预期输出：**
```
Already up to date.
或
Updating files...
```

---

### 场景：修改代码并同步到云端

#### 步骤2：本地修改代码

1. 在VS Code中打开文件：`顺势而为+激进+做T.py`
2. 进行修改（例如调整参数）
3. 保存文件（Ctrl+S）

#### 步骤3：云端同步

```powershell
# 1. 查看修改了哪些文件
git status

# 预期输出：
# modified:   顺势而为+激进+做T.py

# 2. 查看具体修改内容
git diff

# 3. 添加修改的文件
git add 顺势而为+激进+做T.py
# 或添加所有修改
git add .

# 4. 提交修改（写上清晰的说明）
git commit -m "优化了买入条件参数"

# 5. 推送到GitHub
git push origin main

# 6. 确认推送成功
# 预期输出：
# Writing objects: 100% (3/3), 325 bytes | 325.00 KiB/s, done.
```

### 场景：在其他电脑上继续工作

# 1. 进入项目目录

## 常用命令速查

### 查看状态
```powershell
git status              # 查看修改状态
git diff                # 查看修改内容
git log --oneline       # 查看提交历史
```

### 提交代码
```powershell
git add .               # 添加所有修改
git commit -m "说明"    # 提交
git push                # 推送到GitHub
```

### 获取代码
```powershell
git pull                # 拉取最新代码
git clone <url>         # 克隆仓库（首次）
```

### 撤销操作
```powershell
git checkout -- 文件名   # 撤销文件修改
git reset HEAD 文件名    # 取消暂存
git log                 # 查看历史，找到commit id
git reset --hard id     # 回退到指定版本
```

## 实际示例

### 示例1：修改策略参数

```powershell
# 1. 修改文件后...

# 2. 查看状态
git status
# Output: modified: 顺势而为+激进+做T.py

# 3. 提交
git add 顺势而为+激进+做T.py
git commit -m "将买入金额从10000改为15000"
git push

# 4. 完成！
```

### 示例2：添加新文件

```powershell
# 1. 创建新文件：新策略.py

# 2. 查看状态
git status
# Output: Untracked files: 新策略.py

# 3. 添加并提交
git add 新策略.py
git commit -m "添加了新的策略文件"
git push
```

### 示例3：删除文件

```powershell
# 1. 删除文件

# 2. 提交删除
git add -A
git commit -m "删除了临时文件"
git push
```

## VS Code图形化操作

### 方法1：使用源代码管理面板

1. 点击左侧源代码管理图标（Ctrl+Shift+G）
2. 查看修改的文件列表
3. 点击"+"暂存修改
4. 输入提交信息
5. 点击"✓"提交
6. 点击"..." -> "推送"

### 方法2：使用命令面板

1. Ctrl+Shift+P 打开命令面板
2. 输入 "Git: "
3. 选择相应命令

## 常见问题

### Q1: 提示 "Please tell me who you are"

```powershell
# 配置Git用户名和邮箱
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### Q2: 提示 "failed to push some refs"

```powershell
# 先拉取最新代码，再推送
git pull origin main
git push origin main
```

### Q3: 冲突解决

```powershell
# 1. 拉取时出现冲突
git pull

# 2. 手动编辑冲突文件，解决冲突

# 3. 添加解决后的文件
git add 冲突文件

# 4. 提交
git commit -m "解决了合并冲突"
git push
```

### Q4: 忘记提交信息

```powershell
# 修改最后一次提交
git commit --amend -m "新的提交信息"
git push --force  # 如果已推送过，需要强制推送
```

## 最佳实践

1. **经常提交**：小步快跑，不要积累太多修改
2. **写好说明**：清晰的提交信息方便以后查看
3. **先pull再push**：避免冲突
4. **定期备份**：重要修改及时提交
5. **使用分支**：大功能开发使用分支，避免影响主分支

## 快捷方式

创建批处理文件快速同步：

```powershell
# sync.bat
@echo off
echo 正在同步代码...
git add .
git commit -m "更新代码 %date% %time%"
git push
echo 同步完成！
pause
```

双击运行即可快速同步！
